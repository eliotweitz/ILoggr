iLoggr supports three major types of logging in your application:

1. You can create and increment counters that can report the number of times a particular event occurs
2. You can log a string or numerical value.
3. You can log the current location of the phone.

Counters can be useful for keeping track of how often a condition occurs whether it is a feature of a game that has been selected or an error condition that needs to be tracked.

More conventional logging can be done at any point of the iPhone application that can record string or numerical (double) data. This information can be automatically augmented with the phone location (lat, lon) if location has been enabled.
The following code added to your application will log a custom generated in your application: 

    // Record an exceptional condition
    NSString* eDesc = [NSString stringWithFormat:@"Exception in drawing routine drawShape: %@", [ex stringValue]];
    [ILoggr record:eDesc callback:nil];

This line of code sends a message to the ILoggr server validating the application ID and recording a launch for this application on this phone. Since all calls to the back end server are asynchronous, a callback can be included to call when the operation is completed.

Once this code is deployed and in use, you can produce graphical reports that show the number of unique users for your application over a particular time period and also a breakdown of locations for the users (if enabled).

The following code shows how you can log a data value: 

    // Record a value
    [ILoggr recordWithData:@"chats sent" data:chatCount callback:nil];

If the iLoggr library was initialized using the "launchWithLocationEnabled" method, both of the example method calls above would have also logged the phone's latitude and longitude along with the string and data values if the user opted into providing their location.
Below is a line of code that will increment an application counter: 
  // Increment the "reached level 3" counter
    [ILoggr incrementCounter:@"Level3Counter" callback:nil];
This code will increment a counter named "Level3Counter" by 1. This could be useful for tracking how many time a level of a game has been reached.
