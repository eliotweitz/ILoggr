To use the features and code with iLoggr, you must download and link your application code with the iLoggr static libraries: iloggr.a and iloggrsim.a in your XCode iPhone application project.

Instructions for install:

There are two versions of the iloggr library for different platform targets, iPhone simulator (iloggrsim.a) and iPhone/iPod (iloggrsim.a).

These libraries are currently compiled using the iPhone 3.1 Beta 3 release.

Download the zip file containing iloggr.a, iloggrsim.a and iloggr.h. Unzip, and save all three files to a folder on your hard drive. In XCode, add a group called "iLoggr" to your project at the top level and copy all three files into that group. It is most convenient to have both libraries copied into your XCode project so when you build for either different simulator or device targets, one will link and the other will be ignored.

Please note that the following frameworks are required to successfully link the ILoggr library (even if location capabilities aren't used by your application):

UIKit.framework
CoreLocation.framework

Linking in the CoreLocation framework should have no affect on your app's performance if you are not using the location capabilities of iLoggr.

iLoggr.plist File

Once you have created your new application using this website, you need to download and include into your iPhone XCode project the iloggr.plist file associated with your application. There is a link to download this file in the "Application Summary" panel.

This file contains two values that identify both your account and your application's unique ID to the iLoggr server. For security purposes, do not release this file to anyone outside of your development team.

Minimal logging example
The following code added to your application will allow tracking of the number of unique iPhone users and the duration of use in a report: 

- (void)applicationDidFinishLaunching:(UIApplication *)application {    
    // Record launch of application
    [ILoggr launch:nil];
}

This line of code sends a message to the ILoggr server validating the application ID and recording a launch for this application on this phone. Since all calls to the back end server are asynchronous, a callback can be included to call when the operation is completed.

Once this code is deployed and in use, you can produce graphical reports that show the number of unique users for your application over a particular time period and also a breakdown of locations for the users (if enabled).

The following code added to your application will allow tracking of the usage duration for your application: 

- (void)applicationWillTerminate:(UIApplication *)application {    
    // Record the duration of the application use
    [ILoggr exit];
}

This will log the total time of usage for the application and can be used to generate duration analysis graphs.
